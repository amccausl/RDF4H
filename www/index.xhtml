rdf4h: RDF for Haskell

rdf4h is the beginnings of a library for working with RDF in Haskell. I'm making it available at this very early stage of development in order to motivate myself to not be too distracted by other projects. At present, it consists of just an N-Triples parser and some basic functions for querying an RDF graph.

The N-Triples parser is stable for the moment. When Parsec is byte-string able (a Google SoC project), I'll revisit the parser and convert it to use byte-strings. It does handle all the W3C testcases correctly, but I haven't tested it much otherwise (read 'at all').

There is a core module which contains fundamental types such as Graph, Triple, Node and friends. The graph representation is the simplest thing that would work initially, and will change very soon.

There is API documentation available here, and you can pull the Darcs repository from the URL of this page if you want the latest and greatest. Patches are welcome.
